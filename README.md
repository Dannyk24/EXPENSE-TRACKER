# Expense Tracker 💰

A simple and intuitive Expense Tracker application designed to help users manage their daily spending, track financial habits, and gain better control over their money.

---

## 📌 Project Overview

This project allows users to:

* Add income and expenses
* Categorize transactions
* View a list of all transactions
* Filter and sort transactions

The goal was to build a clean, functional system while improving my backend logic, data handling, and overall application structure.

---

## 🚀 Features

* Add, edit, and delete transactions
* Categorize expenses (e.g., Food, Transport, Utilities)
* Real-time balance calculation
* Simple and responsive UI
* Persistent data storage (database/local storage depending on implementation)

## ⚠️ Challenges & Lessons Learned

One of the biggest challenges I faced during this project was **poor initial planning**.

At the start, I jumped straight into coding without fully mapping out:

* The system structure
* Design patter
* User flow and interactions and especially the transactions page logic

This led to:

* Frequent rewrites of code
* Confusing logic and messy structure
* Difficulty scaling or adding new features

### 💡 What I Learned

This experience taught me the importance of **proper project planning and system design** before writing code.

I improved my approach by:

* Breaking the project into smaller components
* Designing the data flow before implementation
* Structuring my backend and routes more clearly
* Thinking ahead about scalability and maintainability

As a result, I developed a stronger understanding of:

* System design fundamentals
* Clean code organization
* Planning before execution
